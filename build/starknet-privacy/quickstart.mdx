<Note>
This quickstart uses explicit `--url` flags to avoid network ambiguity in local runs.
</Note>

## 1. Start local simulation network (Devnet)

`starknet-devnet` is a **local simulation tool**, not a full Starknet node.

```bash
starknet-devnet --seed 0 --host 127.0.0.1 --port 5050
```

## 2. Build contracts

```bash
cd starknet-privacy
scarb build
```

## 3. Configure account and network explicitly

```bash
sncast account list
# or create/import according to README

export STARKNET_RPC_URL=http://127.0.0.1:5050
```

Use `--url "$STARKNET_RPC_URL"` (or `--network`) in every `sncast` command.

## 4. Declare and deploy (pattern)

```bash
sncast --url "$STARKNET_RPC_URL" declare --contract-name PrivacyPool
sncast --url "$STARKNET_RPC_URL" deploy --class-hash <CLASS_HASH> --constructor-calldata <CALLDATA>
```

## 5. Register viewing key and run first private flow

Use the exact script or test entry listed in the repo README. In most repos this is either:

- an integration test (for example `snforge test -- e2e`), or
- a CLI command from a Rust crate.

If an SDK is introduced later, add it to this step (generate proof -> submit actions) and to [Install and build](/build/starknet-privacy/installation).

## 6. Verify

- notes/nullifiers updated on-chain,
- expected events emitted,
- paymaster reimbursement (if used).
