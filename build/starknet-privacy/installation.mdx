The [**starknet-privacy**](https://github.com/starkware-libs/starknet-privacy) repository README is the **source of truth** for pinned versions and exact commands. Use the steps below as the **expected stack**; copy version numbers and any extra steps from the README when they differ.
</Warning>

## 1. Clone the repository

```bash
git clone https://github.com/starkware-libs/starknet-privacy.git
cd starknet-privacy
```

## 2. Cairo / Starknet toolchain (contracts)

The privacy pool and proof logic are written in **Cairo**. Install the same toolchain as in [Environment setup](/build/quickstart/environment-setup):

**macOS / Linux** — Starkup (Scarb, Starknet Foundry, Starknet Devnet):

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.starkup.sh | sh
```

Verify:

```bash
scarb --version
snforge --version && sncast --version
starknet-devnet --version
```

Use the **Scarb / Cairo versions required by the repo** (see `Scarb.toml` and the README)—if the project pins an older or newer Scarb, switch with `asdf` per the [appendix](/build/quickstart/appendix) or the README.

## 3. Rust (client prover)

Client-side proofs use **Stwo** over Cairo execution. If the repository ships a **Rust** workspace for the prover or CLI:

1. Install [Rust](https://rustup.rs/) (`rustup`).
2. Use the **Rust toolchain version** specified in `rust-toolchain.toml` or the README (e.g. `rustup default 1.xx`).

```bash
# Example once README specifies the crate/workspace layout
cargo build --release
```

## 4. Build Cairo packages

From the repository root (or the path given in the README):

```bash
scarb build
```

## 5. Run tests

```bash
snforge test
```

If the README documents integration tests, scripts, or a different test entrypoint, follow that instead.

## 6. Optional — Node / pnpm

If the repo includes TypeScript or script tooling:

```bash
# Example — use the package manager named in the README
npm install
# or: pnpm install
```
